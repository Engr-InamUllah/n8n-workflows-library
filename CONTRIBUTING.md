# Contributing

1. Open an issue describing the outcome and category.
2. Fork and create a focused branch.
3. Use `templates/workflow-documentation-template.md`.
4. Number after the current highest ID and use lowercase kebab-case.
5. Include all required documentation sections.
6. Never include secrets, credential exports, customer data, or raw executions.
7. Parse and import-test JSON examples in a supported n8n version.
8. Submit a pull request with compatibility and validation details.

Contributions are released under the MIT License.
