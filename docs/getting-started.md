# Getting Started

Use a current supported n8n release, a test workspace, and non-production app accounts.

1. Choose a blueprint by business outcome.
2. Map trigger, validation, processing, destination, and failure path.
3. Create credentials in n8n Credentials.
4. Build or import while inactive.
5. Test success, empty, duplicate, rate-limit, and failure cases.
6. Add timeouts, retries, deduplication, and an Error Trigger workflow.
7. Activate with least privilege and monitor early runs.

Markdown files are implementation blueprints. JSON files in `examples/` are importable demonstrations, not production-ready integrations. Provider node parameters can vary by n8n version.
